# Promotion Cluster Protocol

When promoting a cluster of related fleeting notes, do not collapse the cluster into one oversized concept note.

## Process

1. Identify the durable vocabulary.
2. Identify durable claims.
3. Identify bridges between ideas, but keep only bridges that capture a transformation, translation, equivalence, representation shift, or conceptual transfer.
4. Identify open tensions or contradictions, but keep only durable two-sided tensions or trade-offs.
5. Explicitly record when no bridge or no contradiction/tension note is warranted.
6. Create or update an index note if the cluster has more than five durable notes.
6. Preserve worked examples as example/project notes or references, not as generic concepts.

## Expected result for a technical cluster

A technical cluster should usually produce:

- several atomic concept notes;
- one or more permanent claim notes;
- one bridge note if the cluster contains a non-obvious conceptual translation or transformation;
- an index note if it opens a topic area.

Do not create weak bridges merely because concepts connect to another topic. Put ordinary relationships in link context or an index note.

Do not create weak contradiction notes merely because the topic is difficult. Contradiction/tension notes should preserve a real trade-off, disagreement, limitation, ambiguity, or apparent conflict.

Do not create one mega-note called `Automatic differentiation` that contains all details about JVP, VJP, dual numbers, pullbacks, and reverse mode.
