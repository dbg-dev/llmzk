# Promotion Cluster Protocol

When promoting a cluster of related fleeting notes, do not collapse the cluster into one oversized concept note.

## Process

1. Identify the durable vocabulary.
2. Identify durable claims.
3. Identify bridges between ideas.
4. Identify open tensions or contradictions.
5. Create or update an index note if the cluster has more than five durable notes.
6. Preserve worked examples as example/project notes or references, not as generic concepts.

## Expected result for a technical cluster

A technical cluster should usually produce:

- several atomic concept notes;
- one or more permanent claim notes;
- one bridge note if concepts connect to another topic;
- an index note if it opens a topic area.

Do not create one mega-note called `Automatic differentiation` that contains all details about JVP, VJP, dual numbers, pullbacks, and reverse mode.
